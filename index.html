I. API 概述

介绍 - 尖端的真人娱乐场、老虎机和迷你游戏 API 提供商
在一个将彻底改变在线游戏行业的开创性举措中，已正式推出其备受期待真人娱乐场、老虎机和迷你游戏 API 平台。这个最先进的解决方案承诺为玩家和运营商提供无与伦比的游戏体验。

1.1 无缝钱包 API

运营商应在他们那边提供无缝钱包集成 API。将在玩家下注或获胜时调用这些方法，并应更新他们的余额。
  1. 单一钱包/无缝钱包需要您实现这3个API
    1. 查询玩家余额API，必须实现
      1. /api/balance
    2. 交易API，必须实现
      1. /api/transaction
    3. 批量交易API（捕鱼游戏用到，如果没有捕鱼游戏可以不用实现）
      1. /api/batch-transactions
  2. 后台配置好单一钱包回调地址
    1. 配置方式：登录后台 → 右上角的profile → 配置并保存单一钱包回调
    2. 如果配置的回调地址为：https://www.google.com
      1. 查询玩家余额API完整地址是：https://www.google.com/api/balance
      2. 交易API完整地址是：https://www.google.com/api/transaction
      3. 批量交易API完整地址是：https://www.google.com/api/batch-transactions
  3. 获取token
    1. /auth/createtoken
    2. 需要拿到的token缓存起来复用，用于后续其他API的请求
  4. 获取游戏链接
    1. /game/launch-url
    2. 进入游戏
  5. 获取游戏注单记录
    1. /betting/history/by-date-v2
方法/部分 描述 状态
Balance 返回玩家的余额 必需
Transaction 使用此方法，系统将通知赌场运营商有关游戏交易，如下注、获胜。 必需

1.2 余额转账 API

运营商可以使用此 API 来打开游戏并将资金转入玩家在系统中的余额。转账钱包基本对接流程如下
  1. 获取token
    1. /auth/createtoken
    2. 需要拿到的token缓存起来复用，用于后续其他API的请求
  2. 创建玩家
    1. /user/create
    2. 用于在我方创建玩家账号
  3. 存款
    1. /user/deposit
  4. 获取游戏链接
    1. /game/launch-url
    2. 打开链接即可进入游戏
  5. 获取游戏注单记录
    1. /betting/history/by-date-v2
方法/部分 描述 状态
CreateUser 此方法允许在端注册新玩家 必需
Balance 返回玩家的余额 必需
Deposit 此方法将资金转入玩家的余额 必需
Withdraw 此方法从系统内的玩家余额中转出资金 必需
WithdrawAll 此方法从系统内的玩家所有余额中转出资金 必需

1.4 API 速率限制策略

为确保公平使用并防止滥用，我们的 API 对每个端点强制执行速率限制。每个 API 端点都有自己的限制，在特定时间段后重置。如果用户超过限制，API 将返回 429 Too Many Requests 错误，进一步的请求必须等待直到重置期。速率限制通过响应头传达：
  • X-RateLimit-Limit – 时间窗口内允许的最大请求数
  • X-RateLimit-Remaining – 达到限制前剩余的请求数
  • X-RateLimit-Reset – 限制重置的 Unix 时间戳
端点 限制 时间窗口
POST /auth/createtoken 5 30 秒
POST /betting/history/by-date-v2 2 10 秒
POST /game/users/batch-rtp 1 3 秒

2.1 创建令牌

使用此方法，赌场运营商将获得身份验证承载令牌。此 API 允许每 30 秒最多 5 个请求。超过此限制可能会导致临时限制或请求被拒绝。注意：过多的令牌生成请求可能导致账户被阻止。
POST /auth/createtoken

请求参数

名称 类型 描述 状态
clientId string 这是代理 clientId，您可以在代理页面找到 必需
clientSecret string 这是代理 clientSecret，您可以在代理页面找到 必需

响应参数

名称 类型 描述
token string(250) 这是用于请求承载令牌的 token
expiration long 此令牌的过期时间戳（秒） 
POST /api/v2/auth/createtoken HTTP/2
Host: xxx.com
content-type: application/json
accept: */*
Request Body
{
    "clientId": "testId",
    "clientSecret": "testSecret"
}

Response:
{
    "token": "eyJhbGciOiJIUzUxMiI........... SlDhiXdB2y5Q",
    "expiration": 1716257131
}

IV. 常见问题 (FAQ)

一般问题

问：什么是？
答：是一个完整的全球 API 服务，用于集成在线赌场、老虎机和迷你游戏。它使运营商能够以安全且可扩展的方式管理游戏、玩家、钱包和报告。
问：谁可以使用？
答：想要将赌场、老虎机和迷你游戏内容集成到其平台的在线赌场运营商、聚合商和开发人员。

集成

问：如何开始集成？
答：请求访问仪表板。一旦获得批准，您将收到 API 凭据（ClientId 和 ClientSecret）。在生产之前，请从身份验证指南开始。
问：是否支持无缝钱包和转账钱包？
答：是的。两者都可以。如果是无缝钱包模式，客户端必须注册回调 URL 以用于回调 API。
问：是否需要加白IP？
答：是的。测试&正式环境都需要加白IP，IP加白只能向我们申请，请留意最多只能加白20个IP

游戏

问：有哪些类型的游戏可用？
答：老虎机、迷你游戏、真人荷官桌、轮盘、百家乐和体育投注集成。
问：新游戏多久添加一次？
答：根据我们的计划和客户的要求，新的老虎机和赌场游戏每月和每周发布。
问：是否限制运营国家区域？
答：不限制
问：请求收到401的响应是空的，是什么原因？
答：有可能有两个原因：1.请求API的IP未加白或者提供的IP不是实际请求的IP（请留意IPV4/IPV6）,2.clientSecret输入错误，和clientId不匹配
问：是否支持修改下注限额？
答：支持，但是需要协助提供对应的参考网站链接，账号密码，供我方技术进行登录同步修改

安全与合规

问：有多安全？
答：所有 API 请求都需要 HTTPS 和基于令牌的身份验证。还支持 IP 白名单。您可以在个人资料页面上重置 API 凭据。所有代理必须提供白名单 IP 才能使用我们的 API。
问：是否遵守游戏法规？
答：是的。我们提供详细的审计日志、公平游戏验证和特定司法管辖区的合规报告。

交易

问：支持哪些货币？
答：支持多货币操作，包括法币和加密货币，具体取决于您的设置。我们使用 ExchangeRate-Api 获取实时法币汇率，使用 coingecko api 获取实时加密货币汇率。所有可用货币的汇率可以在货币汇率页面找到。
问：支持哪些语言？
答：支持多语言操作，遵循客户的要求。请参阅支持的语言部分。所有代理在发送启动 URL 请求时都可以使用语言参数。默认为 en。语言代码参数格式在这里。
问：是否有最大交易限制？
答：为确保公平使用并防止滥用，我们的 API 对每个端点强制执行速率限制。请参阅 API 速率限制策略部分。

RTP

问：什么是RTP？
答：RTP 是 Return to Player 的缩写，中文通常翻译为 “玩家回报率” 或 “返奖率”。在博彩和电子游戏（如老虎机、捕鱼机）中，RTP 是一个核心的理论数值，用来衡量一款游戏在长期运行下，会将投注总额的百分之多少返还给玩家。
问：通过RTP怎么计算大概理论值的下注和派彩总流水？

                            以下为假设理论值，实际值可能会有所不同
                            RTP = 90% = 0.9
                            假设本金 = 100
                            假设每次下注 = 1
                            不考虑下注上限、不考虑爆分封顶（纯理论）
                            
                            理论值的下注总流水 = 假设本金 / (1 - RTP) = 100 / (1 - 0.9) = 1000
                            理论值的派彩总流水 = 理论值的下注总流水 * RTP = 1000 * 0.9 = 900
问：我可以实时控制用户的 RTP 吗？
答：是的，除了所有真人娱乐场桌，Aviator，体育游戏 之外，所有老虎机和迷你游戏都可以。在线平台运营商可以单独将用户 RTP 应用于单个提供商或单个用户，应用的效果会立即反映。
问：我可以通过 API 完成所有 RTP 设置吗？
